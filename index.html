Privacy Policy

Last updated: July 13, 2026

This Privacy Policy explains how Player2TV, developed and operated by Player2Apps, handles user data.

Important Content and Service Disclaimer

Player2TV is strictly a media player utility. The app does not provide, host, include, sell, lease, promote, or distribute any media content, live channels, movies, series, playlists, accounts, access credentials, or streaming subscriptions.

The app is empty upon installation. Users must provide their own legally acquired playlist URLs or login credentials from their own chosen service providers or servers.

Player2TV does not process payments, in-app purchases, subscriptions, or license-key purchases inside the app.

No Account Creation

Player2TV does not allow users to create an account with Player2Apps. The app does not provide user registration, email/password sign-up, social login, or user profile accounts.

If the user uses Backup/Restore, the app generates an anonymous restore code. This restore code is used only to recover backup data and does not create a Player2Apps user account.

Data We Collect and Use

Player2TV collects and uses limited data only for core app functionality, playback, app security, backup/restore, and service reliability.

  • Playlist information, such as playlist URL, server URL, playlist name, and playlist type. This is used to load the user’s own configured media source.
  • Account credentials, such as username and password, when the user enters Xtream Codes or M3U login details. These credentials are stored locally on the device and are used to connect to the server selected by the user. If the user chooses Backup/Restore, saved playlist credentials may be included in the backup so they can be restored on another device.
  • Hashed Android ID, used to generate or maintain a stable device identifier for app security, backup/restore, device recognition, and connection management. The Android ID is hashed using SHA-256 before being sent to Player2Apps servers.
  • App version, package name, APK hash, and app integrity data, used to verify that the app version is supported and to protect the service against tampered or outdated builds.
  • Device/session status data, such as online status, recent heartbeat time, app version, IP address, and device identifier. IP address may be processed and temporarily stored in server logs or session records to establish connections, prevent abuse, and support connection management.
  • Backup data, when the user chooses to use Backup/Restore. Backup may include saved playlists, playlist names, server URLs, usernames, passwords, hidden/favorite channel settings, category order, app preferences, and other restorable settings. Backup is user-initiated and used only to restore settings on the same or another device.
  • Playback and app preferences, such as favorites, hidden channels/categories, subtitle settings, stream format preference, EPG settings, data saver preference, recent items, and layout preferences. These are stored locally unless included in a user-created backup.

How Data Is Used

Collected data is used only for the following purposes:

  • Loading playlists and media sources configured by the user.
  • Authenticating with servers entered by the user.
  • Maintaining saved playlists and app settings.
  • Providing backup and restore functionality when requested by the user.
  • Checking app version and app integrity.
  • Displaying active device/session status.
  • Improving app reliability, caching, and playback performance.
  • Managing feature availability returned by the server.
  • Preventing abuse and protecting app functionality.

Background Data Use

Player2TV may perform limited background work, such as refreshing cached playlist data or maintaining app data needed for faster loading. Background activity is limited to app functionality and is not used for advertising, tracking, or profiling.

Backup and Restore

Backup/Restore is optional and user-initiated. When the user creates a backup, Player2TV sends backup data to Player2Apps servers and returns an anonymous restore code. The restore code can be used to recover saved settings and playlists on another device.

Backup data is retained only as long as necessary to provide restore functionality and may expire automatically. Users may request deletion of backup data by contacting us.

Third-Party Servers and User-Configured Services

Player2TV allows users to connect to servers, playlist URLs, and media sources that they provide themselves. When users enter a third-party server URL or playlist, the app sends the required requests directly to that server to load channels, movies, series, EPG data, and playback streams.

Player2Apps is not responsible for the privacy practices, availability, content, or legality of third-party servers configured by users.

Third-Party SDKs and Libraries

Player2TV does not include third-party advertising SDKs, analytics SDKs, or tracking SDKs. The app does not use your data for advertising or marketing profiling.

Data Sharing

We do not sell, rent, trade, or share personal data with advertisers or data brokers.

Data may be sent only to:

  • The server or playlist URL manually configured by the user.
  • Player2Apps servers for app integrity checks, online session status, feature availability, and optional backup/restore.
  • Service providers only when necessary to operate hosting, security, or backup functionality.

Data Storage and Retention

Most app settings and playlist information are stored locally on the user’s device using Android storage mechanisms such as SharedPreferences and local databases.

Server-side app session records, hashed device identifiers, IP logs, and backup records are retained only as long as needed for app functionality, security, abuse prevention, restore support, or legal compliance.

Security

Player2TV uses industry-standard encryption such as HTTPS to secure data transmitted between the app and Player2Apps servers where supported.

Some user-provided playlists or streams may use HTTP if the user’s selected provider only supports HTTP. Device identifiers are processed as hashes where applicable.

Users should only enter playlist URLs and account credentials from sources they trust and are authorized to use.

User Control and Data Deletion

Users can delete locally stored app data at any time by clearing app data in Android settings or uninstalling the app.

To request deletion of backup data, hashed device identifiers, IP/session records, or other server-side records stored by Player2Apps, contact us using the email below. If the data belongs to a third-party server configured by the user, the user should contact that third-party server provider directly.

Children

Player2TV is not directed to children and does not knowingly collect personal data from children. Users are responsible for ensuring that any playlist or media source they add is appropriate and legally permitted.

Changes to This Policy

We may update this Privacy Policy from time to time. Updates will be posted on this page with a revised “Last updated” date.
