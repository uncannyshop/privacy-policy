Privacy Policy for Player2TV

Last updated: July 17, 2026

This Privacy Policy explains how Player2TV, operated by Player2Apps, handles user data. It applies to the Player2TV Android and Android TV app and related Player2TV server features.

Content and Service Disclaimer

Player2TV is a media player and playlist management app. The app does not sell media content, process payments, offer in-app purchases, or provide Google Play subscriptions.

Users must only add or use playlists, login details, streams, and EPG sources that they are authorized to access. Player2TV does not host or control third-party media sources that users configure.

When a user enters a third-party playlist, server URL, or stream URL, requests may be sent directly to that source so the app can load and play the user's selected media.

Accounts

Player2TV does not provide self-service account registration inside the app. The app does not include email/password sign-up, social login, or user profile creation.

If a user receives access credentials from an app operator or service administrator, those credentials are used only for app access, playlist access, connection limits, feature availability, and related support.

Data We Collect

Player2TV collects and uses only data needed for app functionality, service reliability, app security, connection management, and optional backup/restore.

  • Playlist and source information: playlist URL, server URL, playlist name, playlist type, EPG URL, and stream format preference.
  • Authentication information: playlist username and password or account credentials entered by the user to access a configured playlist or service.
  • Device identifiers: a generated stable device identifier and a SHA-256 hash of the Android ID, used for app security and connection management. Player2TV does not collect the device's real hardware MAC address.
  • App integrity and security information: package name, app version, APK hash, signing certificate hash, app token validation status, and supported-build checks.
  • Device and session status: IP address, User-Agent, online/offline status, first seen time, last activity time, active connection records, and current stream status when needed to enforce connection limits or troubleshoot playback.
  • App preferences and playback settings: favorites, hidden channels or categories, category order, EPG settings, subtitle settings, layout preferences, recent items, and data-saver preferences.
  • Backup data: only when the user chooses backup/restore. Backup may include saved playlists, server URLs, usernames, passwords, EPG URLs, favorites, hidden items, category order, app preferences, and other restorable settings.
  • Metadata requests: movie/series metadata, logos, and poster images when the app loads information from the user's configured source or from metadata/image providers used by the app.

How We Use Data

We use collected data for the following purposes:

  • To load playlists, EPG data, media metadata, and streams configured by the user.
  • To authenticate with servers or services selected by the user or app operator.
  • To save and restore user settings, playlists, and preferences.
  • To provide optional backup/restore and return a restore code when server backup is used.
  • To verify app version, package name, signing status, and app integrity.
  • To enforce active-device and connection limits.
  • To provide feature availability and entitlement settings returned by the Player2TV server.
  • To maintain playback reliability, caching, diagnostics, and abuse prevention.
  • To respond to support, privacy, or deletion requests.

We do not use user data for advertising, ad targeting, or marketing profiling.

Backup and Restore

Backup/Restore is optional and user-initiated. Player2TV may support local backup to the device and server backup through Player2Apps servers.

When server backup is used, backup data is uploaded and a restore code is returned so the user can restore settings on the same or another device. Server backup records are intended to be temporary and may expire automatically. The current default backup expiration period is 30 days unless configured differently on the server.

Users may request deletion of backup data by contacting us.

Third-Party Sources and Services

Player2TV lets users connect to playlist URLs, server URLs, streams, EPG URLs, images, and metadata sources. These may be operated by third parties.

Requests sent to user-configured third-party sources are necessary for playback and playlist loading. Those third parties may receive request information such as IP address, User-Agent, stream URL parameters, or login details required by that source.

Player2Apps is not responsible for the privacy practices, availability, content, or legality of third-party sources configured or used by the user. Users should only use sources they trust and are authorized to access.

Third-Party SDKs and Libraries

Player2TV does not include third-party advertising SDKs, analytics SDKs, or tracking SDKs. The app uses functional libraries for playback, networking, image loading, background work, and Android UI. These libraries are used to operate the app and are not used by Player2Apps for advertising or user profiling.

Data Sharing

We do not sell, rent, trade, or share personal or sensitive user data with advertisers or data brokers.

Data may be sent only to:

  • Player2Apps servers for app integrity checks, account or playlist access, active-device monitoring, connection-limit enforcement, feature availability, diagnostics, and optional backup/restore.
  • User-configured playlist, EPG, metadata, image, or streaming servers when needed to load or play the user's selected content.
  • Hosting, security, or infrastructure service providers that process data only as needed to operate Player2TV.
  • Authorities if required by law, legal process, or to protect the rights, safety, and security of users, Player2Apps, or the service.

Data Storage and Retention

Most settings and playlist information are stored locally on the user's device using Android app storage. Server-side account, device, connection, app monitor, IP/session, and backup records are retained only as long as needed for app functionality, service operation, security, abuse prevention, support, backup/restore, or legal compliance.

Users can delete locally stored app data by using the app's delete/reset options where available, clearing app data in Android settings, or uninstalling the app. Server-side deletion requests can be sent to the contact address below.

Security

Player2TV uses reasonable security measures to protect data, including HTTPS for Player2Apps production server communication where configured, certificate pinning for configured Player2Apps endpoints, access controls, and server-side protections.

Some user-provided playlists, streams, images, or EPG sources may use HTTP or other security practices controlled by the user's selected provider. Users should only enter sources and credentials from providers they trust.

Children

Player2TV is not directed to children and is not intended for use by children. We do not knowingly collect personal data from children. Users are responsible for ensuring that any playlist or media source they add is appropriate and legally permitted.

Data Deletion Requests

To request deletion of server-side data associated with Player2TV, including backup records, account/device records, hashed device identifiers, or session records, contact us at the email below. We may need enough information to locate the records, such as the username, restore code, or device identifier shown in the app.

Some limited records may be retained if required for security, fraud prevention, dispute resolution, or legal compliance.

If the data belongs to a third-party playlist, streaming, EPG, metadata, or account provider, users should contact that provider directly.

Changes to This Policy

We may update this Privacy Policy from time to time. Updates will be posted on this page with a revised "Last updated" date.
